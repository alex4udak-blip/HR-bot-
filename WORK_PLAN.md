## О проекте

**Enceladus** — enterprise HR-платформа: Telegram-бот + веб-админка. Собирает данные о кандидатах (текст, голос, видео), анализирует через Claude API, ведёт базу кандидатов, матчит с вакансиями, детектит дубликаты.

**Проблема:** Команда использует 5+ разных инструментов (ClickUp, Factorial, ручной труд). Нет единого трекинга проектов и ресурсов.

**Цель:** Превратить Enceladus в единую платформу с тремя точками доступа (блоками) на одном домене.

---

## Архитектура платформы — 3 блока

```
Enceladus (один домен, одна кодовая база)
├── /hr          — HR-блок (рекрутинг, воронки, Настя)
├── /projects    — Project Management + Task Management (трекинг проектов, Jira/Tempo-лайт)
└── /analytics   — Аналитика по всему (проекты, HR, ресурсы)
```

Три блока имеют **разные точки доступа**, но живут на одной платформе.

### Что нужно

**Проекты:**

**Ресурсы:**

- Не жёсткий тайм-трекинг (не дрочить часы)
- Примерная оценка усилий на проект

## ПРИОРИТЕТ 2 — HR-блок (для Насти, замена ClickUp + Factorial)

### 2.1 "Волшебная кнопка" — Chrome Extension

Рекрутер открывает профиль на HH/LinkedIn/Habr → нажимает кнопку → CV парсится в Enceladus.

**Сайты:** HeadHunter, LinkedIn, Habr Career

**Как работает:**
1. Расширение определяет страницу профиля/резюме
2. Парсит данные (имя, контакты, опыт, навыки, образование)
3. Отправляет в API Enceladus
4. Создаётся кандидат со статусом "Новый"

**Референс:** Huntflow

> **Олег уже начал работу над расширением** (Лёха отправил)

### 2.2 Воронки кандидатов (канбан)

**Статусы:**
- Новый → Отбор → Собеседование назначено → Собеседование пройдено → Оффер → Вышел на работу
- Отказ (на любом этапе)
- Прошёл практику / Не прошёл

**Функционал:**
- Drag & drop между статусами
- Фильтрация по вакансии, рекрутеру, дате
- История переходов

### 2.3 Приём анкет
- Форма для кандидатов → попадает в воронку как "Новый"

### 2.4 Кнопка "Пригласить в Prometheus"
- Автописьмо на email с ссылкой на регистрацию

## ПРИОРИТЕТ 3 — CRM сотрудников (аналог Factorial)

### 3.1 База сотрудников
- Карточка: ФИО, должность, отдел, дата выхода, контакты, документы
- Автоимпорт из воронки (кто прошёл до конца)

### 3.2 Оргструктура
- Отделы, команды, кто у кого руководитель
- Визуальная оргсхема

### 3.3 Шаблоны документов + электронная подпись
- Шаблоны с переменными: `{{firstName}}`, `{{position}}`, `{{startDate}}`
- Автоподтягивание из карточки сотрудника
- Графическая подпись в интерфейсе (не КЭП)
- Автотриггеры: после 3 мес. испытательного → отправка договора

**Референс:** Factorial, Bamboo HR

### 3.4 Личные кабинеты
- Сотрудник видит свои данные, документы, баланс отпусков

### 3.5 Отпуска
- Запрос → одобрение HR → списание дней

### 3.6 Роли и права
- Суперадмин (Александр, Виктор) → Лиды → HR (Анастасия) → Сотрудники
- Permission Sets для точечного расширения прав

Единая платформа Enceladus с тремя блоками:
