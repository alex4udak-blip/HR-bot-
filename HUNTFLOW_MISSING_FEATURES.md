# Huntflow missing features

Фиксируем элементы интерфейса Huntflow, которые нужны для визуального сходства, но не имеют полноценного аналога в текущем HR-bot. Их не реализуем как функционал без отдельной задачи.

- HR sidebar: `Календарь` есть в Huntflow, отдельной HR-страницы календаря в HR-bot сейчас нет. Пункт не показываем, пока нет реального раздела.
- Candidate actions: `СМС` в Huntflow открывает промо/подключение SMS-рассылок и напоминаний. В HR-bot SMS-модуль сейчас отсутствует, поэтому полноценный SMS-попап не добавляем без отдельной задачи.
- Candidate actions: `Форма обратной связи` и `Анкета кандидата` в Huntflow являются отдельными сценариями коммуникации/сбора данных. В HR-bot есть общий конструктор форм, но нет привязанного к карточке кандидата сценария отправки Huntflow-анкеты/feedback-формы, поэтому эти кнопки дают безопасную заглушку без записи данных.
- Candidate bulk actions: массовая `Отправка` кандидатам в Huntflow является отдельным коммуникационным сценарием. В HR-bot есть массовые операции по кандидатам/заявкам, но нет Huntflow-сценария массовой коммуникационной отправки из этого меню, поэтому пункт оставлен только как безопасная заглушка.
- Interview modal: расширенные Huntflow-контролы `Другое`, выбор timezone, Zoom/тип видеовстречи, выбор прикрепляемого резюме, дополнительные участники, напоминания, email-уведомления и проверка доступности требуют отдельных календарных/интеграционных моделей. В HR-bot есть обработка Zoom/Meet-созвонов в модуле звонков, но нет создания видеовстречи и проверки доступности из HR-интервью; здесь сохраняются только дата, время, место, комментарий и приватность интервью, остальные контролы являются безопасными заглушками.
- Resume actions: для демо-резюме на HR-странице нет исходного файла, поэтому Huntflow-кнопки `Скачать` и `Распечатать` не могут выполнить файловую операцию без отдельного backend/file-flow.
- HR settings center: `Темы оформления`, `API и вебхуки`, `Почта и календарь`, `Джоб-сайты`, `Интеграции` в Huntflow являются отдельными настройками. В HR-bot нет полноценного пользовательского UI для этих разделов, поэтому в центре настроек они показаны как недоступные без добавления нового функционала.
- HR sidebar plus menu: `Добавить событие` есть в меню Huntflow, но отдельного глобального создания HR-события без выбранного кандидата/воронки в HR-bot сейчас нет. Пункт не показываем, пока нет реального сценария.
- Vacancy form: `Настроить поля формы`, `Сколько человек нужно нанять`, дедлайн найма, `Этапы подбора` как отдельная настройка формы, `Шаблон оффера` и прикрепление файлов к вакансии в Huntflow являются отдельными настройками вакансии/воронки. В HR-bot нет отдельных моделей для этих параметров/файлов вакансии, поэтому они переносятся визуально или мапятся на ближайшие существующие поля без добавления нового backend-функционала.
- Vacancy request preview: кнопка Huntflow `Не брать в работу` требует отдельного прямого сценария отказа от назначенной заявки. В HR-bot есть `Взять в работу` (`/{vacancy_id}/take`) и косвенное скрытие через `dismissed_by`, когда рекрутёр закрывает/отменяет уже созданный клон заявки, но нет отдельного endpoint/UI-действия для отказа от оригинальной назначенной заявки; поэтому кнопку не показываем, пока такой сценарий не будет добавлен.
- New candidate modal: `Импорт из почты` в Huntflow открывает импорт кандидата из подключенной почты. В HR-bot нет отдельной интеграции почтового импорта кандидатов, поэтому кнопка переносится визуально без добавления нового backend-функционала.
- New candidate modal: `Загрузка из файла -> Из xlsx` в Huntflow ведет к отдельному импорту кандидатов из таблицы. В HR-bot есть отдельная страница CSV-импорта и парсинг резюме через parser flow, но нет xlsx-import сценария именно из модалки нового кандидата.
- Vacancy detail: Huntflow предлагает инструкцию/действие для сохранения найденных на джоб-сайтах резюме прямо в текущую вакансию. В HR-bot есть глобальная страница `/extension` и backend-flow magic-button, но текущий блок внутри карточки вакансии не является полноценным Huntflow-действием для этой вакансии, поэтому там оставлена безопасная заглушка.
- Vacancy detail: меню `options-main/options-2` в Huntflow содержит дополнительные настройки отображения/действия вакансии. В HR-bot сейчас есть редактирование/закрытие/удаление через существующие формы, но нет отдельного Huntflow-меню с полным набором действий.
- My vacancies toolbar: в Huntflow на поиске вакансий есть расширенные фильтры, сортировки и дополнительные представления. В HR-bot на `/my-funnels` сейчас поддержаны только реальные операции: поиск по названию, фильтр статуса и админский фильтр по рекрутеру. Остальные Huntflow-контролы не показываем, пока нет backend/API и продуктового сценария.
- Closed vacancies search: в Huntflow строки закрытых вакансий могут показывать заказчиков и кандидатов с `Оффер принят`. Текущий список вакансий HR-bot не отдаёт эти агрегированные связи в vacancy overview, поэтому экран использует только реальные поля вакансии: даты, департамент, рекрутер и количество кандидатов.
- Closed vacancies filters: в Huntflow меню `Фильтры` для закрытых вакансий поддерживает расширенные поля поиска по заказчикам/офферам и другим агрегатам. В HR-bot сейчас нет этих полей в API списка вакансий, поэтому реализован только безопасный попап со сбросом доступных фильтров без добавления фиктивных данных.
- All candidates top stages: в Huntflow есть отдельные этапы `Принятие решения/ СБ`, `Вышел на работу`, `Резерв` и онбординг-этапы после оффера. В текущей модели HR-bot есть только реальные статусы `new`, `screening`, `practice`, `tech_practice`, `is_interview`, `offer`, `hired`, `rejected`; поэтому неподдержанные Huntflow-этапы убраны из верхнего меню и смены этапа, без фальшивого маппинга на чужие данные.
- Vacancy detail top stages: в Huntflow внутри вакансии отдельно видны `Интервью с заказчиком` и `Принятие решения/ СБ`. В текущей модели воронки HR-bot эти этапы не представлены отдельными backend-статусами, поэтому верхние этапы вакансии показывают реальные backend-стадии.
